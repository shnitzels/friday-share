Config 2026: פיגמה הופכת את הקנבס למקום שבו עיצוב, קוד, תנועה ו-AI נפגשים

הקינוט של Figma לא היה עוד רשימת פיצ׳רים. הוא היה הצהרת כיוון: הקנבס כבר לא אמור להיות רק המקום שבו מסדרים פריימים, אלא סביבת עבודה שמחברת קוד, אפקטים, תנועה, כלים פנימיים וסוכני AI — בלי להוציא את היצירה מהמרחב המשותף של הצוות.

מקור: Config 2026 Keynote — Figma
יוצר: Figma
אורך: 79:30 דקות
הערת מקור: ל-YouTube לא היו כתוביות זמינות בזמן העבודה; המאמר מבוסס על תמלול מקומי בעזרת Whisper מהווידאו הציבורי.

דילן פילד פתח את Config 2026 עם משפט שמסכם את כל הקינוט: פעם נאמר לנו שהעתיד שייך למי שיודע לקודד. היום, לטענתו, קוד כבר אינו ההפך מעיצוב — הוא חומר גלם עיצובי. כמו צבע, טקסטורה, תנועה או טיפוגרפיה, גם קוד צריך להיות משהו שאפשר למשש, לשכפל, להשוות, לשנות ולשתף על הקנבס.

זו תפיסה חשובה, כי היא שונה מאוד מגל כלי ה-AI לעיצוב שראינו בשנתיים האחרונות. רוב הכלים ניסו להאיץ נקודה אחת בתהליך: לייצר מסך, לכתוב קומפוננטה, לעשות אפקט, ליצור תמונה. Figma מנסה לומר משהו רחב יותר: אם AI אכן מוריד את סף הכניסה, התפקיד של כלי העיצוב הוא להעלות את התקרה — לאפשר לצוותים לעבוד על חומרים חדשים בלי לאבד את הדבר שפיגמה בנתה עליו את החברה: עבודה משותפת, חיה, על קנבס אחד.

השורה התחתונה: Config 2026 מציג את Figma כמעין “מערכת הפעלה ליצירה”: Code Layers לקוד על הקנבס, Weave לכלי AI יצירתיים, פלאגינים גנרטיביים, שיידרים, Figma Motion, וסוכן Figma שמתחיל לצאת מעבר לקובץ העיצוב אל FigJam ו-Slides.

המסגרת: חומרים חדשים וכלים חדשים

Figma חילקה את ההכרזות לשני צירים: חומרים חדשים שאפשר ליצור איתם, וכלים חדשים שמאפשרים לעצב את החומרים האלה. זה נשמע כמו ניסוח שיווקי, אבל הוא עוזר להבין את הקינוט. Code, Motion, Shaders, Workflows, Plugins — כולם מוצגים לא כיעדים נפרדים אלא כעוד סוגי חומר שנכנסים לקנבס.

פילד גם חזר על רעיון שהוא כבר אמר בעבר: AI “מוריד את הרצפה” — כלומר מאפשר ליותר אנשים להתחיל ליצור. אבל הוא הודה שהשאלה האמיתית היא האם AI גם “מעלה את התקרה”. במילים אחרות, האם הוא מאפשר לאנשים חזקים לעשות עבודה שאי אפשר היה לעשות קודם, או רק מייצר ממוצע מהיר יותר. הקינוט כולו ניסה לענות על השאלה הזאת דרך דמואים.

קוד כחומר עיצובי לא רק handoff למפתחים, אלא שכבה אינטראקטיבית שנמצאת על הקנבס.
יצירה כ-workflow לא פרומפט בודד, אלא מערכות חוזרות של רפרנסים, איטרציות, consistency וכלים פנימיים.
תנועה ואפקטים Motion, Shaders ו-3D נכנסים לתוך Figma במקום להישאר בכלים נפרדים.

Code Layers: קוד נכנס פיזית לקנבס

ההכרזה הראשונה והכנראה הכי אסטרטגית הייתה Code Layers. ניקו, אחד מאנשי Figma, הציג את הבעיה כך: סוכני קוד הפכו את הכתיבה של אלפי שורות קוד לקלה יותר, אבל העבודה סביב הקוד עדיין אינה בנויה לחקירה משותפת. כל אחד נמצא “מאות פרומפטים עמוק” עם הסוכן שלו, ובסוף שולחים הקלטת מסך כדי לנסות להסביר מה נבנה.

Code Layers מנסים להפוך את זה למשהו פיגמאי: שכבות קוד שנמצאות על הקנבס, שאפשר להריץ, להשוות, לשכפל, לגרור, לפתוח, ולראות לצד שכבות עיצוב רגילות. אפשר ליצור Code Layer מעיצוב קיים באמצעות “build this with code”, להשתמש בכלי קוד חדש בסרגל הכלים, או לייבא Repository מ-GitHub או תיקייה מקומית ישירות לקנבס.

הדמו הראה כמה דברים מעניינים במיוחד. אפשר להריץ כמה codebases נפרדים במקביל על הקנבס כדי להשוות רעיונות. אפשר לפתוח את הצ׳אט של השכבה ולראות את ההיסטוריה המשותפת — מי ביקש מהסוכן מה, איך הגיעו לתוצאה, ומה השתנה. ואפשר לייצר branch בצורה הכי פיגמאית שיש: מחזיקים Option וגוררים את השכבה.

למה זה חשוב: זה ניסיון להפוך Vibe Coding מפעולה פרטית של אדם מול סוכן לפעולה קבוצתית. אם קוד הופך לשכבה שאפשר לראות, לשכפל ולהשוות, צוותי מוצר מקבלים מרחב משותף לחקירה — לא רק Pull Request בסוף.

אחד הרגעים החזקים בדמו היה המעבר מקוד לעיצוב וחזרה. ניקו הקליט Flow מתוך שכבת קוד, Figma חילצה ממנו מצבים סטטיים כעיצוב, ואז המשתתפים ערכו את אותם states בכלי העיצוב הרגילים. לאחר מכן “Update Code Layers” ניתח את השינויים ועדכן את הקוד. מכיוון שהשכבה מחוברת ל-GitHub, אפשר גם לדחוף את השינויים חזרה לריפו.

זה עדיין לא נשמע כמו פתרון מלא ל-production engineering, אבל כ-direction זה חזק: הקו בין prototype, design state ו-implementation מתחיל להיטשטש. Code Layers יושקו ב-closed beta, עם onboarding שמתחיל ביולי, ויהיו זמינים למשתמשי Full Seat בכל התוכניות.

Weave: לא “תמונה יפה”, אלא workflow יצירתי חוזר

החלק הבא עסק ב-Weave, החברה שפיגמה רכשה ושילבה בפלטפורמה. איתי שף, ממייסדי Weave, תיאר בעיה שכל מי שעבד עם GenAI ויזואלי מכיר: קל ליצור משהו מעניין, קשה ליצור משהו שימושי לפרויקט אמיתי. עבודה יצירתית מקצועית אינה פרומפט אחד; היא דורשת רפרנסים, איטרציות, עקביות, פידבק, ולעיתים גם כלי עבודה פנימיים שמותאמים לצוות.

Weave מבוססת על workflows ויזואליים, בסגנון node-based. צמתים יכולים להכיל תמונות, וידאו או מודלים תלת־ממדיים, או לבצע פעולות באמצעות מודלי AI וכלים ידניים כמו color grading וטיימליינים. הדמואים כללו יצירת סרטוני 360 למוצרים מתמונות קדמיות ואחוריות, רינדור מהיר של אנימציה בסגנון מסוים, ו-style transfer שמאחד תמונות מוצרים ממקורות שונים לשפה חזותית אחת.

החידוש ב-Figma הוא שה-workflows האלה מגיעים אל הקנבס ככלים. המשתמש לא צריך לראות את “הנודלס” של כל הגרף; הוא בוחר כלי מהרשימה, מסמן inputs ו-style reference, ומקבל output בתוך הקובץ. בהמשך, צוותים יוכלו לארוז workflows משלהם ככלים פנימיים ולשתף אותם דרך Tools Panel.

הנקודה האסטרטגית: Figma לא מנסה רק להוסיף “Generate Image”. היא רוצה שכל צוות יוכל להפוך את השפה החזותית שלו לכלים חוזרים — כאלה שמייצרים consistency ולא רק וריאציה אקראית.

Generative Plugins: כל אחד בונה את הכלי הקטן שחסר לו

אחר כך הגיעו Georgia ו-Raji עם הכרזה שמתחברת היטב ל-Weave: החל מ-Config 2026 אפשר לייצר פלאגינים בעזרת Figma Agent. אלו אינם הפלאגינים הקלאסיים שדורשים Desktop App, קוד ופרסום לקהילה. הם חיים בתוך הקובץ, מופיעים ב-Properties Panel, אפשר להשתמש בהם ולשתף אותם ישירות, והם נבנים בכמה פרומפטים.

הדמו התחיל מצורך קטן ומוכר: באפליקציית אודיו היו מלבנים שטוחים, והמעצבת רצתה להפוך אותם ל-waveform אורגני. היא ביקשה מהסוכן ליצור פלאגין שמרנדמז את הגובה של תוכן הפריים כך שייראה כמו waveform אמיתי. בלי לכתוב קוד, נוצר כלי קטן עם controls, והיא יכלה להריץ אותו, לשנות פרמטרים ולשפר.

הדוגמאות הבאות היו אפילו יותר מעניינות כי הן הראו את הרעיון של פלאגין אישי או צוותי: פלאגין שמייצר Bento grids בסגנון של מעצב ספציפי, פלאגין שמייצר מסלולי מרוצים אמיתיים על בסיס data, ופלאגין לעיצוב ציפורניים באסתטיקה של Config. הרעיון הוא לא שכל אלה “חשובים” בפני עצמם, אלא שהסף לבניית כלי קטן יורד כמעט לאפס.

יש כאן שינוי תרבותי: במקום לבקש מצוות פלטפורמה לבנות utility קטן, או לוותר כי “זה רק חוסך חמש דקות בשבוע”, מעצב יכול לנסות לבנות אותו בעצמו. ואם הכלי שימושי, הוא נשאר עם הקובץ או הצוות. זה סוג חדש של אוטומציה יצירתית: לא מערכת ענקית, אלא הרבה כלים קטנים ומקומיים.

Shaders: אפקטים שהיו שמורים למי שמבין מתמטיקה

אחד החלקים הכי צבעוניים בקינוט היה ההכרזה על Shaders ב-Figma. Raji הסביר ששיידרים כבר קיימים מאחורי הקלעים של רינדור הקנבס — למשל ב-blur, glass effects או gradients — אבל עד עכשיו היה קשה מאוד ליצור אותם בעצמך. עכשיו אפשר ליצור Shader Fills ו-Shader Effects בעזרת Figma Agent.

Shader Fill הוא חומר המקור: צבעים, gradients, images ודברים שממלאים שכבה. Shader Effect יושב מעל החומר ומשנה אותו: מעוות, מכופף, מטשטש, ממזג, מוסיף lens distortion, chromatic aberration, page curl, gooey merge, interactive particles ועוד. הדמו הדגיש שהכול נשאר שכבות Figma אמיתיות, לא תמונה שטוחה.

החלק החשוב הוא האינטראקציה. לשיידרים יש sliders ו-on-canvas handles, כך שאפשר לשנות אותם באופן ישיר. אפשר גם לערוך אותם דרך הסוכן, להוסיף reference image, לבקש אפקט “שנראה כמו זה”, ואז להמשיך בלולאת edit. Effects יכולים להיות stackable: משנים את הסדר, מקבלים תוצאה אחרת, ומתחילים לשחק כמו עם blend modes — רק הרבה יותר עמוק.

זמינות: יצירת פלאגינים, Shader Effects ו-Shader Fills עם הסוכן זמינה למשתמשי Full Seat בתוכניות בתשלום. Interactive Shaders יגיעו בהמשך אחרי ש-Figma תשלים אופטימיזציות ביצועים, ובהמשך תהיה גם אפשרות לפרסם לארגון ולקהילה.

Figma Motion: טיימליין הופך לפרימיטיב עיצובי

ההכרזה הגדולה השנייה הייתה Figma Motion, שנולדה מהצוות של Modify. פיגמה מציגה את Motion לא כתוסף צדדי, אלא כפרימיטיב חדש בתוך Figma Design. יש Timeline, Keyframes, Easing, Presets, Styles, Variables ו-Components — וכל זה עובד עם שאר המערכת של Figma.

הדמו התחיל בפעולה פשוטה: נכנסים ל-Motion Mode, לוחצים Space והעיצוב מתעורר לחיים. כדי להוסיף אנימציה מפעילים Auto Key עם Shift+K, מזיזים אובייקט, מוסיפים rotation או scale, ומנגנים. אפשר לערוך easing בזמן שהאנימציה מתנגנת, לגרור keyframes, לשנות תזמון, ולראות הכול על הקנבס.

גם כאן הסוכן הוא לא “קסם סופי” אלא accelerator. Piers ביקש מה-Agent להוסיף לכל התמונות fade in, לטקסטים slide-and-fade, ואז fade out כדי ליצור loop. התוצאה הייתה עריכה מלאה על הטיימליין — לא וידאו סגור. כשה-transition out היה איטי מדי, הוא פשוט בחר keyframes ושינה תזמון.

אחד הדמואים היפים היה Plugin שנבנה כדי לפתור פעולה מייגעת בטיימליין: stagger של אובייקטים. במקום להזיז ידנית בלוקים של keyframes, הפלאגין מאפשר לבחור אובייקטים, להגדיר מרווח, להפוך סדר, ואפילו לסדר כניסה לפי Bezier curve. לפי המציג, הכלי נבנה בארבעה או חמישה פרומפטים.

Figma גם הציגה Preview של 3D Transforms: אנימציה של X, Y ו-Z, עם controls תלת־ממדיים על הקנבס. זה עדיין לא ההכרזה המלאה, אבל המסר ברור: Motion לא מיועד רק למיקרו־אינטראקציות קטנות, אלא לשפה ויזואלית עשירה יותר.

החיבור החשוב: Motion + Shaders + Design Systems

החלק הכי מעניין ב-Figma Motion אינו עצם קיומו של טיימליין. After Effects, Principle, Rive וכלים אחרים קיימים שנים. מה שמעניין הוא החיבור של Motion לכל מה שכבר נמצא ב-Figma: קומפוננטות, comments, collaboration, variables, dev mode, MCP, וכעת גם shaders.

בדמו, comments הופיעו גם על הטיימליין; spotlight למשתמש אחר הציג את המיקום שלו בתוך האנימציה; שינוי בקומפוננטה התעדכן בזמן אמת בסצנה; ו-Shader Effects יושמו על גבי תוכן מונפש. כלומר, אפקטים לא היו “בייקד” לתוך וידאו, אלא חיו מעל שכבות Figma אמיתיות והגיבו לתנועה שמתחתיהן.

פיגמה הראתה גם איך לוקחים את זה לייצור. אם רוצים לבנות, עוברים ל-Dev Mode, מחלצים קוד ישירות, או שולחים דרך Figma MCP ל-LLM או ל-Figma Make. ואם לא רוצים קוד, אפשר לייצא MP4, WebM, Animated SVG או GIF. Motion יוצא ל-beta היום בכל התוכניות למשתמשי Full Seat.

למה זה משנה למוצר: אם motion נשאר בכלי נפרד, הוא הופך מהר מאוד לנספח: קובץ וידאו, הסבר למפתח, או reference. אם הוא חי בתוך מערכת העיצוב, עם רכיבים, comments ו-handoff, הוא יכול להפוך לחלק אמיתי מהשפה המוצרית.

Figma Agent מתרחב: Connectors, Skills, FigJam ו-Slides

לקראת הסוף פילד חזר ל-Figma Agent, שהחל להתגלגל לכל המשתמשים יום לפני הקינוט. הסוכן תומך ב-attachments וב-connectors שמושכים context מכלים כמו Notion, Slack, Granola, Hex, GitHub ו-Atlassian. כלומר, הוא לא אמור לעבוד רק על מה שנמצא בקובץ, אלא להביא לתוך העיצוב מידע ממערכות העבודה של הצוות.

הוכרזו גם Custom Skills: אפשר להפוך פרומפטים שחוזרים על עצמם לפקודות slash חוזרות. פילד נתן דוגמאות כמו בדיקות compliance, יצירת synthetic user test, או אוטומציה של workflow צוותי. זו נקודה מעניינת במיוחד כי היא מחברת בין Agent לבין ידע ארגוני: לא רק “תעשה לי X”, אלא “תעשה את X בדרך שבה הצוות שלנו עושה X”.

ולבסוף הגיע tease להמשך: FigJam Agent ו-Slides Agent. פילד תיאר אפשרויות כמו תרגום codebase לדיאגרמות מובנות, הפיכת רעיונות ל-prioritization ו-next steps, או הפיכת customer briefs למצגות pitch deck ממותגות. כלומר, הסוכן של Figma מתחיל כעוזר עיצוב, אבל הכיוון הוא עוזר עבודה רחב יותר לכל תוצר ויזואלי ושיתופי.

מה זמין עכשיו ומה מגיע בהמשך

  • ללא צריכת קרדיטי AI בזמן הבטא: לפי פילד, העדכונים שהוצגו לא יצרכו AI credits בזמן שהם ב-beta.
  • Closed beta / בקרוב: Code Layers, FigJam Agent ו-Slides Agent.
  • הרשמה מוקדמת: Figma הפנתה לכתובת figma.com/config-betas למי שרוצה לנסות את הבטאות הסגורות.
  • Config הבא: Figma הכריזה שגם Config 11 ייצא לדרך מאוחר יותר השנה, בבנגלור, הודו, ב-15 באוקטובר.

הקריאה בין השורות: Figma רוצה להיות המקום שבו ה-AI נהיה צוותי

הסיפור הגדול של Config 2026 הוא לא “Figma הוסיפה AI”. זה כבר קרה. הסיפור הוא ש-Figma מנסה לפתור בעיה עמוקה יותר של עידן הסוכנים: עבודה יצירתית עם AI נוטה להיות פרטית, מפוזרת ולא שקופה. כל אחד עובד בצ׳אט שלו, עם context שלו, ומביא תוצאה בסוף. זה מנוגד ל-DNA של Figma, שבנתה את עצמה סביב קנבס משותף.

לכן כמעט כל הכרזה בקינוט מנסה להחזיר את ה-AI למרחב משותף: Code Layers מציגים קוד על הקנבס; Generative Plugins נשארים בקובץ; Weave Tools אורזים workflows לצוות; Shader ו-Motion נשארים עריכים; הצ׳אטים של הסוכן נשמרים וניתנים ללמידה; ו-Connectors מביאים context חיצוני לתוך הקובץ.

זו גם הסיבה שהמשפט “Code is material for design” כל כך חשוב. פיגמה לא רוצה שהמעצב יהפוך למפתח במובן הצר. היא רוצה שכל חומר שמרכיב מוצר דיגיטלי — קוד, תנועה, אפקט, data, workflow, כלי פנימי — יהיה משהו שאפשר לעבוד עליו יחד, ויזואלית, באותה סביבת עבודה.

המסר הכי מעניין: AI לבדו מוריד את הרצפה. הקנבס המשותף אמור להעלות את התקרה. אם Figma תצליח, היתרון שלה לא יהיה “הסוכן הכי חכם”, אלא היכולת להפוך תוצרים של סוכנים לחומרים משותפים, עריכים, וניתנים להמשך עבודה.

מה לקחת מזה כמעצב, PM או בונה מוצר

  • פרוטוטייפים עומדים להתקרב לקוד: Code Layers ו-Make מראים כיוון שבו חקירה, עיצוב ויישום מתקרבים זה לזה.
  • כלים פנימיים קטנים יהפכו לנורמה: Generative Plugins ו-Skills מאפשרים לצוותים לבנות אוטומציות ספציפיות בלי להפוך כל צורך לפרויקט הנדסי.
  • Consistency תהיה קריטית יותר מ-generation: Weave מדגישה שהערך אינו ביצירת asset בודד, אלא בבניית workflow חוזר שמייצר שפה עקבית.
  • Motion ואפקטים יעברו שמאלה בתהליך: במקום להגיע בסוף כ-polish, הם יכולים להפוך לחלק ממערכת העיצוב וה-handoff.
  • הפער בין כלי AI פרטיים לבין עבודה צוותית יגדל: מי שיצליח לחבר AI לשפה, לקבצים ולזיכרון של הצוות יקבל יתרון אמיתי.

Config 2026 היה קינוט עמוס, אולי אפילו עמוס מדי. אבל מתחת לדמואים הצבעוניים הייתה טענה פשוטה: העתיד של כלי העיצוב לא יהיה עוד חלון צ׳אט שמייצר דברים. הוא יהיה קנבס שבו כל סוגי החומר — קוד, תנועה, ויזואליה, נתונים, כלים וסוכנים — יכולים להיפגש, להתערבב, ולהפוך למוצר יחד.

פילד סיים בקריאה לקהל: “AI יכול להוריד את הרצפה. זה תלוי בכם להעלות את התקרה.” אם צריך לסכם את Config 2026 במשפט אחד, זה בדיוק זה: Figma מנסה לתת ליוצרים יותר חומרים ויותר מנופים, אבל להשאיר את השאלה החשובה אצלם — מה תבנו עכשיו כשאפשר לבנות כמעט כל דבר?
