עקומת היכולות: איך לבנות למודלים שהולכים להיות טובים בהרבה

ג׳רמי מאנתרופיק מציג את עקומת ההתקדמות של Claude כמתכנת: יותר קוהרנטיות במשימות ארוכות, פחות קוד מיותר, יכולת טובה יותר להבין מפרטים, והצורך לבנות מערכות שלא ננעלות על מגבלות המודל של היום.

מקור: The capability curve · מבוסס על תמלול מקומי מלא של ההרצאה לאחר שחזור האודיו מהסרטון

השורה התחתונה

ההרצאה ממסגרת את Claude Code כיכולת שנמצאת על עקומה חדה. הדוגמאות מראות לא רק שיפור באיכות התוצאה, אלא שינוי בסגנון העבודה: המודל כותב פחות שורות כדי להשיג יותר, מבין טוב יותר מה המשתמש רוצה, ומחזיק משימות ארוכות בלי להתפרק באותה מהירות.

אחת הבעיות שהוא מתאר היא שכחה במשימות מורכבות. כשנותנים מפרט עמוס בעשרות דרישות, מודלים קודמים נטו לפספס פרטים או לאבד את הכיוון. ההתקדמות היא ביכולת לשמור קוהרנטיות לאורך ריצות agent ארוכות, לתכנן לפני ביצוע ולהישאר צמוד למטרה.

הלקח למפתחים חד: אם המערכת שלכם בנויה סביב הנחות של מודל חלש, היא תתיישן מהר. צריך לבנות כך שאפשר להחליף מודל, להריץ evals על המשימות האמיתיות, ולתת למערכת לנצל מודלים חזקים יותר ברגע שהם מגיעים.

החלק המעניין ביותר הוא self-improvement: לאפשר לסוכנים לבחון את הפלט של עצמם, להריץ בדיקות, לשפר prompts וכללים, ולעזור לכם לשפר את הסוכן עצמו. זה לא קסם אוטונומי מלא, אלא לולאת עבודה שבה Claude Code הופך גם לכלי בנייה וגם לכלי ביקורת.

נקודות לקחת הלאה

  • המודלים משתפרים לא רק באיכות קוד אלא בקוהרנטיות לאורך זמן.
  • מפרטים ארוכים עדיין דורשים תכנון, בדיקות ואימות.
  • מערכת AI טובה צריכה להיות model-upgradable כבר מהיום הראשון.
  • Evals צריכים למדוד את המשימות האמיתיות ולא benchmark כללי.
  • הכיוון הוא סוכנים שעוזרים לשפר את הסוכנים עצמם.
