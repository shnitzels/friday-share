1. מה זה LangAlpha ולמה זה שונה

כלי AI פיננסיים זה לא חדש. Bloomberg GPT, FinGPT, Kensho — כולם ניסו לחבר בין AI לשוק ההון. הבעיה? הם מטפלים בהשקעה כמשימה חד-פעמית: שאל שאלה, קבל תשובה, גמור.

אבל השקעות אמיתיות הן Bayesian. אתה לא מתחיל בידיעה מלאה — אתה מתחיל בתזה, מעדכן אותה כשנתונים חדשים מגיעים, מחדד פוזיציות, ובונה הבנה עמוקה שמצטברת לאורך שבועות וחודשים.

"כל כלי AI פיננסי היום מטפל בהשקעה כ-One-Shot: שאל שאלה, קבל תשובה, המשך. אבל השקעה אמיתית היא Bayesian — אתה מתחיל בתזה, נתונים חדשים מגיעים מדי יום, ואתה מעדכן את האמון שלך בהתאם."

ה-Insight של LangAlpha: אם Claude Code הצליח כסוכן תכנות בגלל שהוא מבין Codebase כשלמות — אותו עיקרון חל על מחקר פיננסי. תן לסוכן Workspace מתמשך, וה-Research מצטבר ומשתפר לאורך זמן.

2. הבעיה הגדולה שאף AI פיננסי לא פתר: MCP Tools וגודל ה-Context

אחת הבעיות הטכניות המרכזיות שהצוות נתקל בהן — וכתבו עליה בפוסט ה-Show HN שלהם:

"MCP tools לא ממש עובדים עבור נתונים פיננסיים בקנה מידה. קריאת Tool אחת עבור חמש שנים של מחירים יומיים שופכת עשרות אלפי Tokens לתוך חלון ה-Context. וספקי נתונים ארוזים בעשרות כלים ב-MCP Server אחד — ה-Schemas בלבד יכולים לאכול 50,000+ Tokens לפני שהסוכן עשה בכלל משהו שימושי."

הגדלה: ספק נתונים פיננסי כמו Polygon.io מספק אלפי Endpoints שונים — מחירי מניות, נתוני אופציות, נתוני Macro, רווחים רבעוניים, תיקי SEC EDGAR, ועוד. אם מנסים להכניס את כל ה-Schema הזה ל-Context של המודל — לא נשאר מקום לחשיבה בפועל.

3. Programmatic Tool Calling (PTC) — הפתרון שמשנה הכל

זהו החידוש הטכנולוגי המרכזי של LangAlpha. במקום שהסוכן יקרא לכלים ישירות ויקבל JSON שנשפך ל-Context — הסוכן כותב קוד Python שרץ ב-Sandbox מבודד ומחזיר רק את התוצאה הסופית.

📌 איך זה עובד:

1. המודל כותב קוד Python

2. הקוד נשלח ל-Sandbox של Daytona (ענן מבודד)

3. בתוך ה-Sandbox, הקוד מייבא מודולים שנוצרו אוטומטית מה-MCP Schemas

4. המודולים מתקשרים עם שרתי ה-MCP ב-JSON-RPC

5. שרתי ה-MCP פונים לשירותי ה-API (FMP, Yahoo, Polygon)

6. הנתונים מעובדים, ותוצאה קומפקטית חוזרת למודל

התוצאה: ל-LangAlpha יש ~80 כלים פרוסים על פני מספר שרתי MCP — והעלות ב-Prompt היא זהה לשרת עם 3 כלים ולשרת עם 30. רק שורת תקציר אחת לכל שרת נכנסת ל-Context. הפרטים מגיעים רק כשצריך.

מה זה מאפשר:

• בנייה של DCF Model מלא (Discounted Cash Flow) לחברה ספציפית עם נתוני 5 שנים ויזואליזציה מובנית

• ניתוח Earnings על מחזור של כמה שנים בחציה אחת של Context

• עיבוד תיקי SEC EDGAR בסכאל תעשייתי

• בניית גרפים אינטראקטיביים שמוצגים ישירות ב-UI

4. Workspaces מתמשכים — השקעה היא Bayesian, לא One-Shot

בניגוד לצ'אטבוטים רגילים שמתחילים בשיחה חדשה בכל פעם, LangAlpha בנוי סביב Workspaces. כל Workspace ממופה ל-Sandbox מתמשך עם מבנה ספריות מסודר:

📁 work/<task>/ — שטחי עבודה לכל משימה (נתונים, גרפים, קוד)

📁 results/ — דוחות וניתוחים מגובשים

📁 data/ — מאגרי נתונים משותפים

📄 agent.md — זיכרון המחקר המתמשך של הסוכן

ה-agent.md הוא הלב של המערכת. זהו קובץ שה-Middleware מזריק לכל קריאת LLM, המכיל:

• מטרות ה-Workspace ושאלות פתוחות

• ממצאים מרכזיים מניתוחים קודמים

• אינדקס של Threads ושל קבצים חשובים

בפועל: אתה יוצר Workspace לנושא מחקרי ("Q2 Rebalance", "Data Center Demand Deep Dive", "Energy Sector Rotation") — הסוכן מראיין אותך על המטרות ואסטרטגיית ההשקעה שלך, מייצר ניתוח ראשוני, ושומר הכל. שבוע אחר כך — פותחים Thread חדש, הסוכן מכיר את כל הקונטקסט וממשיך מאיפה שנעצר.

5. Agent Swarm — מחקר מקביל בזמן אמת

אחד הפיצ'רים המרשימים: כשמפעילים Skill מסוים (למשל "Morning Note" — דוח בוקר), הסוכן שולח סוכני-משנה מקביליים לאסוף:

• נתוני שוק בזמן אמת

• חדשות רלוונטיות

• Context מאקרו כלכלי

כל סוכן-משנה רץ עם Context Window מבודד, ערכת כלים מותאמת, ויכולת לקבל עדכונים בזמן ריצה ("Live Steering") — בלי לחכות לסיום הריצה. תוצאה: Morning Note מקיפה עם ויזואליזציות אינטראקטיביות מוטמעות.

6. Skills — Workflows מוכנים לחקר פיננסי

LangAlpha כולל Skills מובנים שמתעוררים ב-Slash Command או בזיהוי אוטומטי של ה-Intent:

📊 DCF Analysis — בניית מודל תזרים מזומנים מהוון

📰 Initiating Coverage — דוח כניסה ראשוני לכיסוי של מניה

💰 Earnings Analysis — ניתוח דוח רבעוני

🌅 Morning Note — דוח בוקר מסוכם עם ויזואליזציות

📄 Document Generation — ייצור מסמכי מחקר מסודרים

הרעיון: Skills מגבילים ומכוונים את הסוכן לעבוד בתוך Workflow מוגדר, ומחשיפים לו רק את הכלים הרלוונטיים. כך מוקטנת הסנטרופיה של מחקר פתוח.

7. ממשק המשתמש: דשבורד + TradingView + WebSocket

ה-Web UI בנוי ב-React 19 + Vite + Tailwind, ומספק:

• גרפים פיננסיים אינטראקטיביים מוטמעים (TradingView integration)

• נתוני שוק בזמן אמת ב-WebSocket

• צפייה בקבצים בפורמטים מרובים (PDF, CSV, JSON, תרשימים)

• ניטור חי של סוכני-משנה בריצה

• שיחות שניתן לשתף

• TUI (Terminal UI) למי שמעדיף לעבוד ב-CLI

• אינטגרציה ל-Slack ו-Discord

8. אוטומציות ו-Price Triggers

אחד הפיצ'רים המפתיעים: Price-Triggered Automations. ניתן להגדיר אוטומציות שמתפעלות כש-מניה או מדד מגיע למחיר ספציפי בזמן אמת. לדוגמה:

• "כשמניית NVDA תגיע ל-$400 — הרץ ניתוח של High-Low breakdown לשנה האחרונה"

• "כשה-VIX יעלה מעל 25 — הפק דוח סקירה של Sector Rotation"

ה-Secretary (סוכן Flash מהיר) מנהל את יצירת ה-Workspace, שולח ניתוחים ברקע, ומחזיר תוצאות בצורה שיחתית.

9. אבטחה: הצפנה, Sandbox, ו-Credential Detection

🔐 הצפנה בסיסמת ה-PostgreSQL pgcrypto — כל API Keys מוצפנים ב-Rest

🔍 Automatic Credential Leak Detection — אם הסוכן בטעות מנסה להדפיס API Key, המערכת מזהה ומסננת

📦 Sandboxed Execution — כל הקוד שהסוכן כותב רץ ב-Daytona Cloud Sandbox מבודד

🔑 Per-Workspace Secret Storage — כל Workspace יכול לשמור Secrets משלו

10. האדריכלות הטכנית

ה-Stack מאחורי LangAlpha:

🐍 Backend: FastAPI — API Routers, WebSocket Proxy, Background Task Manager

🗃️ Database: PostgreSQL (Dual Pool) — App Data + LangGraph Checkpointer

⚡ Caching: Redis — SSE Event Buffer (150K events + reconnect replay), Market Data Cache, Steering Queue

☁️ Sandboxes: Daytona Cloud — בידוד ביצוע קוד

📡 Market Data: FMP, SEC EDGAR, Yahoo Finance, Polygon.io

⚙️ Middleware: 24 שכבות קומפוזיביות — Skill Loading, Plan Mode, Multimodal Input, Auto-Summarization, Context Management

ה-Middleware stack של 24 שכבות הוא שקט אבל קריטי. הוא מאפשר ל-Workflow לרוץ לאורך זמן ארוך בלי לאבד Context, בלי לנפוח את ה-Token Cost, ובלי לפספס עדכונים חיים מהמשתמש.

11. BYOK — הביאו את המודל שלכם

LangAlpha לא נועל אתכם על מודל ספציפי. הוא Agnostic לחלוטין:

• Claude OAuth (Claude Code OAuth) — חברים כבר מנויים ל-Claude? חברו אותו ישירות

• OpenAI OAuth (ChatGPT) — זהה

• Kimi (Moonshot), GLM (Zhipu), MiniMax — מודלים סינים עם תוכניות Coding מוזלות

• BYOK (Bring Your Own Key) — כל API Key שתרצו, מוצפן ב-Rest

Resilience: 3 ניסיונות חוזרים עם Exponential Backoff, ואז Failover אוטומטי למודל גיבוי. רמות ה-Reasoning (Low/Medium/High) מנורמלות אוטומטית בין ספקים.

12. "בנינו את זה כי לא מצאנו אותו"

הפוסט המקורי ב-HN של המפתח (zc2610) כולל את הסיכום הטוב ביותר:

"רצינו שהסוכן יהיה לו Context פיננסי אמיתי כמו שיש ל-Claude Code Context של Codebase — Portfolio, Watchlist, Risk Tolerance, מקורות נתונים פיננסיים — שכולם מוזרקים לכל Call. לפלטפורמות AI Investing הקיימות יש חלק מזה, אבל שום דבר שקרוב למה שסוכן AI אמיתי יכול לעשות. רצינו את שניהם ולא מצאנו, אז בנינו וזה קוד פתוח."

זו נקודת הפתיחה הכי כנה: הם לקחו את הלוגיקה של Claude Code — Agent עם Context של Codebase, כלים מתמשכים, זיכרון Workspace — והחילו אותה על עולם ההשקעות. ה-Result הוא פרויקט עם אדריכלות מרשימה שגם מפתחים וגם משקיעים יכולים להרחיב.
