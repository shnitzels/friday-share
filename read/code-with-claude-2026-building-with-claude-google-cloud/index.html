Building with Claude on Google Cloud: לבנות, לפרוס ולתפעל עם סוכנים

הסרטון חסום לתמלול, ולכן הכתבה מבוססת רק על התיאור הרשמי. הדמו מתאר בנייה מאפס עד deployment תוך 30 דקות, עם Claude, Google Cloud, subagents, MCP ו־skills.

מקור: Building with Claude on Google Cloud · אין תמלול, מבוסס על תיאור רשמי בלבד

השורה התחתונה

הסשן של Claude על Google Cloud מציג את הכיוון המעשי ביותר של הפלייליסט: לא לדבר על סוכנים, אלא לבנות איתם אפליקציה מלאה. לפי התיאור הרשמי, הדמו בונה feedback app מאפס עד deployment תוך שלושים דקות, עם חמישה תפקידים שונים ועם כיסוי של מחזור חיים תוכנתי מלא. הכל נעשה בעזרת Claude ו־Google Cloud, לצד subagents, שרתי MCP ו־custom skills.

אין תמלול זמין, ולכן אי אפשר לשחזר את שלבי הדמו בפועל. אבל התיאור מספיק כדי להבין את הנקודה: Claude מוצג לא ככלי שעונה על שאלה אחת, אלא כשותף בתהליך תוכנה שלם. הוא אמור לעזור בהבנת הדרישות, בתכנון, בקוד, באינטגרציות, בפריסה, בבדיקות ובתפעול. Google Cloud מספקת את התשתית, ו־Claude מחבר בין שכבות העבודה.

מה הסשן מנסה להוכיח

הדמו של אפליקציה עם חמישה roles חשוב כי הוא מדמה מערכת אמיתית יותר מאשר todo app. מערכת עם roles דורשת הרשאות, flows שונים, data model, חוויות משתמש שונות ונקודות כשל שונות. אם Claude יכול לעבוד בתוך סביבה כזו, להשתמש ב־subagents כדי לחלק עבודה, להפעיל MCP כדי לגשת לכלים ולמידע, ולהשתמש ב־skills כדי להחזיק ידע צוותי, אז הוא מתקרב למקום שבו צוותים יכולים להאיץ בנייה אמיתית.

המרכיב החשוב הוא לא רק המהירות. שלושים דקות לדמו זה נחמד, אבל הערך העסקי נמצא ביכולת להפוך ידע חוזר ל־skills, לחבר כלים אמיתיים דרך MCP, ולחלק עבודה בין agents כך שכל אחד מחזיק תחום אחר. זה דומה לארגון קטן של מפתחים בתוך workflow אחד: אחד מתכנן, אחד מייצר קוד, אחד בודק, אחד מטפל בפריסה, ואחד מסכם או מתעד.

לקחים למפתחים

  • Live build טוב צריך להראות את כל מחזור החיים, לא רק code generation.
  • Subagents מועילים כשהמשימה באמת ניתנת לפירוק, למשל UI, backend, data, deployment ו־QA.
  • MCP הוא שכבת חיבור קריטית: בלי כלים ומידע, הסוכן נשאר מנותק מהארגון.
  • Skills הופכים ידע חד־פעמי לתשתית חוזרת. זה ההבדל בין דמו לבין יכולת צוותית.
  • ענן ו־AI מתקרבים: deployment, monitoring ו־permissions צריכים להיות חלק מהשיחה, לא מחשבה מאוחרת.

בהקשר הרחב של הפלייליסט, זהו סשן שמראה איך חזון ה־agentic engineering יורד לקרקע: סביבה אחת שבה Claude לא רק כותב קוד, אלא עוזר להעביר מוצר קטן דרך pipeline שלם עד לפריסה.
